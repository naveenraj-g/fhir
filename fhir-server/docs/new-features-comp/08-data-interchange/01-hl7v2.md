# HL7 v2 Message Parsing

**Medplum reference:** `packages/core/src/hl7.ts`, `packages/server/src/hl7/`

---

## Why HL7 v2?

HL7 v2 (version 2.x) is the most widely deployed healthcare messaging standard globally.  
Despite FHIR's emergence, 95%+ of clinical data exchange still uses HL7 v2:
- **Lab results** arrive as ORU^R01 messages
- **Patient registrations** arrive as ADT^A01 (admit), ADT^A08 (update), ADT^A03 (discharge)
- **Order messages** arrive as ORM^O01 (orders)
- **Scheduling** uses SIU^S12 (appointments)

Without HL7 v2 support, we cannot integrate with any hospital, lab, or legacy system.

---

## HL7 v2 Message Format

HL7 v2 is pipe-delimited text:

```
MSH|^~\&|LAB|HOSP|FHIR|SERVER|20240115103000||ORU^R01|MSG001|P|2.5.1
PID|1||10001^^^HOSP^MR|~Smith^John^A||19850315|M||2106-3|123 Main St^^Boston^MA^02101^USA||(617)555-1234|||M||
OBR|1|ORD001|LAB001|4548-4^HbA1c^LN|||20240115100000|||||||||30001^Smith^Jane
OBX|1|NM|4548-4^Hemoglobin A1c^LN||8.2|%|4.5-6.4|H|||F|||20240115103000
```

### Segments
- `MSH` â€” Message header (source, destination, type, timestamp)
- `PID` â€” Patient identification
- `OBR` â€” Observation request (the order)
- `OBX` â€” Observation result (the value)
- `NTE` â€” Notes and comments
- `EVN` â€” Event type
- `PV1` â€” Patient visit (encounter)
- `AL1` â€” Allergy

---

## Message Types â†’ FHIR Resources

| HL7 Message | Trigger Event | FHIR Resources Created |
|---|---|---|
| `ADT^A01` | Admit patient | `Patient`, `Encounter` |
| `ADT^A08` | Update patient info | Update `Patient` |
| `ADT^A03` | Discharge patient | Update `Encounter` (status=finished) |
| `ADT^A11` | Cancel admit | Update `Encounter` (status=cancelled) |
| `ORU^R01` | Lab result | `Observation`, `DiagnosticReport` |
| `ORM^O01` | Order message | `ServiceRequest` |
| `SIU^S12` | Schedule appointment | `Appointment` |
| `SIU^S15` | Notify appointment cancelled | Update `Appointment` |
| `MFN^M01` | Master file notification | Various reference data |

@router.post("/hl7/v2", operation_id="receive_hl7_message")

### Step 3 â€” HL7 Processing Service

```python
# app/services/hl7_service.py

class HL7Service:
    async def process(self, raw_message: str) -> HL7Result:
        msg = parse_message(raw_message, find_groups=False)
        msg_type = f"{msg.MSH.MSH_9.MSH_9_1.value}^{msg.MSH.MSH_9.MSH_9_2.value}"

        handlers = {
            "ADT^A01": self._handle_admit,
            "ADT^A08": self._handle_update_patient,
            "ADT^A03": self._handle_discharge,
            "ORU^R01": self._handle_lab_result,
            "ORM^O01": self._handle_order,
        }

        handler = handlers.get(msg_type)
        if not handler:
            return HL7Result(status="unsupported", ack=self._build_ack(msg, "AE"))

        resources_created = await handler(msg)
        return HL7Result(status="success", resources=resources_created, ack=self._build_ack(msg, "AA"))

    async def _handle_lab_result(self, msg) -> list[dict]:
        """ORU^R01 â†’ Observation + DiagnosticReport"""
        pid = msg.PID
        patient_id = await self._find_or_create_patient(pid)

        obr = msg.OBR
        report = await self.diag_report_repo.create(
            self._obr_to_diagnostic_report(obr, patient_id), ...)

        observations = []
        for obx in msg.OBX:
            obs = await self.obs_repo.create(
                self._obx_to_observation(obx, patient_id, report.diagnostic_report_id), ...)
            observations.append(obs)

        return [to_fhir_diagnostic_report(report)] + [to_fhir_observation(o) for o in observations]

    def _obx_to_observation(self, obx, patient_id: int, report_id: int) -> dict:
        """Map OBX segment to Observation resource."""
        loinc = obx.OBX_3.OBX_3_1.value
        value = obx.OBX_5.value
        value_type = obx.OBX_2.value  # NM=numeric, ST=string, CWE=coded
        unit = obx.OBX_6.OBX_6_1.value
        range_low, range_high = obx.OBX_7.value.split("-") if "-" in (obx.OBX_7.value or "") else (None, None)
        return {
            "status": "final",
            "code": loinc,
            "subject_id": patient_id,
            "value_quantity": float(value) if value_type == "NM" else None,
            "value_unit": unit,
            "reference_range_low": float(range_low) if range_low else None,
            "reference_range_high": float(range_high) if range_high else None,
            "effective_date": self._parse_hl7_datetime(obx.OBX_14.value),
        }
```

### Step 4 â€” ACK Response

```python
def _build_ack(self, original_msg, ack_type: str) -> str:
    """Build HL7 v2 ACK message."""
    now = datetime.utcnow().strftime("%Y%m%d%H%M%S")
    msg_id = original_msg.MSH.MSH_10.value
    return (
        f"MSH|^~\\&|FHIR|SERVER|{original_msg.MSH.MSH_3.value}|{original_msg.MSH.MSH_4.value}"
        f"|{now}||ACK|ACK{msg_id}|P|2.5.1\r\n"
        f"MSA|{ack_type}|{msg_id}|Message processed successfully\r\n"
    )
```

## MLLP Transport

HL7 v2 messages are typically sent over TCP using MLLP (Minimum Lower Layer Protocol):

```python
# app/hl7/mllp_server.py

MLLP_START = b"\x0B"  # VT (vertical tab)
MLLP_END = b"\x1C\x0D"  # FS + CR

class MLLPServer:
    async def start(self, host: str = "0.0.0.0", port: int = 2575):
        server = await asyncio.start_server(self._handle_connection, host, port)
        async with server:
            await server.serve_forever()

    async def _handle_connection(self, reader: asyncio.StreamReader, writer: asyncio.StreamWriter):
        while True:
            data = await reader.readuntil(MLLP_END)
            if not data:
                break
            message = data[1:-2].decode("utf-8")  # strip MLLP framing
            result = await hl7_service.process(message)
            writer.write(MLLP_START + result.ack.encode() + MLLP_END)
            await writer.drain()
```

## Production Considerations

- Run MLLP server on port 2575 (standard HL7 port) as a separate service
- Store all received HL7 messages in a raw_messages table for audit/replay
- Handle duplicate detection (MSH-10 message control ID)
- Support HL7 v2.3, v2.4, v2.5, v2.5.1 (common in practice)
- MLLP over TLS for security
